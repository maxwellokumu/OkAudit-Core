# Playbook

Use this skill when the user asks for audit playbooks, audit methodology, audit steps, audit procedure, or how to audit.

## Goal
Provide a structured lead IT auditor methodology spanning planning, execution, reporting, and closeout.

## Workflow
1. Use this playbook at the beginning of an engagement or when the user asks for process guidance.
2. If the user asks for one phase only, focus on the requested step.
3. Review README.md for fuller usage examples if needed.
4. Run main.py when the user wants the playbook content rendered directly.
5. Translate the playbook into an actionable next-step checklist for the current audit.

## Inputs
- step (string, optional): Step number to display, or full for the full methodology.

## Bundled Files
- main.py: executable logic for the skill.
- README.md: detailed methodology notes and usage examples.
- skill.yaml: existing repo manifest retained for this project's original packaging.

## Guidance
- Use this skill for methodology and sequencing, not as a substitute for evidence analysis.
- Keep the response procedural and easy to follow.
- When appropriate, direct the user from this playbook into the more specific audit skills.
